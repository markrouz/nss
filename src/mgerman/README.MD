Условие задачи \
На входе два текстовых файла, следующего формата:
@Id
data
@Id
data
...
т.е. нечетные строки содержат некоторые идентификаторы - уникальные в пределах файла, четные строки некоторые данные связанные с идентификатором.
Цель на выходе получить файл, который будет содержать пересечение данных из исходных файлов в формате:
@Id
data1
data2
...
здесь @id - идентификатор, для которого были найдены записи в первом и втором файле, data1 - соответствующая запись data из первого файла, data2 - из второго. Строки для которых идентификатор был найден только в одном файле - игнорируются и не попадают в результаты.
Условия:
- никакого порядка, в котором идут идентификаторы в исходных файлах не задано
- считаем, что файлы большие и загрузить их полностью в память не получится. Допустимо загрузить в память идентификаторы.
- скорость обработки важна - чем быстрее, тем лучше

Пример исходных данных и результата:
source1.txt
@one
foo1
@six
baz1
@four
bar1

source2.txt
@four
foo1
@two
baz1
@six
bar1

result.txt
@six
baz1
bar1
@four
bar1
foo1


Реализованное решение: \
Читаем первый файл, строим хэшмапу айдишник->положение в файле, с которого начинается дата для этого айдишника. Затем читаем второй файл и если айдишник уже есть в мапе то рандом аксессом читаем из первого дату и пишем все в третий файл
